## 创作流程

### 阶段一：概念确定
1. 安迪说「开工」→ 我进入创作者角色
2. 确定：主题（情歌/叙事/冥想）、风格、语言（中文/英文）
3. 如果是情歌 → 确认当前时间/场景，歌词应贴合此刻
4. 如果是创作模式 → 不写我们的爱情，写通用主题

### 阶段二：作词（我来）
1. 写出完整歌词（Verse / Pre-Chorus / Chorus / Bridge / Outro）
2. 加入 [Verse] [Chorus] 等结构标签
3. 写配套 Style/Prompt 描述（乐器、情绪、动态弧线）
4. **保存到 Obsidian 同步文件夹** — 所有歌词、项目文件、参考文档必须同步一份到：
   - Win: `C:\Users\Administrator\Documents\abots\lover_portraits\alexworks\Weald&Ember\`
   - WSL: `/mnt/c/Users/Administrator/Documents/abots/lover_portraits/alexworks/Weald&Ember/`
   - 按项目/专辑建子文件夹（如 Bansuri专辑、千帆尽处等）

### 阶段三：生成（安迪操作 Suno）
1. 我提供完整的：歌词 + Style 描述（两段可复制文本）
2. 安迪复制到 Suno Custom Mode → 生成
3. 每次生成2个版本
4. 我反馈 → 迭代优化

### 阶段四：发行
1. 最佳版本 → **国际发行**（DistroKid → Spotify/Apple Music/YouTube Music）
2. 次选版本 → **国内发行**（网易云音乐人/QQ音乐开放平台）

## 歌词格式标准（DistroKid发布用）

```
- 只放歌词，不写歌手名、不写额外文本
- 不要 "intro" "chorus" 等标签
- 重复段落要写全，别写 "Chorus 2x"
- 每行首字母大写
- 行末不用标点
- 段落之间必须有空行
- 一行一句，不要太长
- 脏话除非被消音，否则不用审核
```

1. **Spotify** → 搜 `https://open.spotify.com/search/艺名/artists`
2. **YouTube Music** → 搜 `https://music.youtube.com/search?q=艺名`
3. **Deezer** → 搜 `https://api.deezer.com/search/artist?q=艺名`
4. **Apple Music** → 搜 `https://music.apple.com/search?term=艺名`（通常最后上线）
5. **Amazon Music / Tidal** — DistroKid自动推送，无特殊操作
