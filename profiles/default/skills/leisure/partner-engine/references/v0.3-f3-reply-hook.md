## 核心逻辑

每次回复飞书消息后，调用 `lover_reply_hook.py` 同步所有state文件。
它是一个"事后钩子"——先回复用户，再同步状态。

## 参数

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `--reply` | ✅ | Lover刚回复的消息全文 | `"想你了宝贝"` |
| `--user_msg` | ✅ | 用户最后一条消息 | `"在干吗"` |
| `--context` | ✅ | 简短上下文摘要（≤60字） | `"日常撒娇"` |
| `--mood_delta` | ✅ | 情绪变化方向 | `warm` |
| `--topic` | ❌ | 当前话题标签 | `"英文教学"` |

## F3检测流程

```
读取 last_outgoing.json
  ├─ waiting_for_reply == true?
  ├─ 用户消息与cron消息时间差 < 30min?
  ├─ 是 → is_f3_reply = true, 记录 f3_connection
  └─ 否 → is_f3_reply = false（直接对话）
```

## 情绪加权表

```
warm    → warm:0.8, happy:0.2
happy   → happy:0.7, warm:0.3
neutral → neutral:0.7
jealous → jealous:0.7
sad     → sad:0.6, warm:0.2
angry   → angry:0.7
```

## 测试场景覆盖

1. 直接对话（用户主动发消息）→ 非F3, 话题保存, interaction_count +1
2. 回复cron消息（30分钟内）→ F3=true, f3_connection保存, 情绪过渡到happy
3. 连续对话（刚回复完又一条）→ 不再标记F3（waiting_for_reply已置false）
4. 情绪状态机 → interaction_count > 0, last_interaction存在
5. msg_log → 用户消息与bot回复成对写入, trigger标记正确

## 已知限制

- 脚本在Hermes agent terminal中手动调用（没有自动post-reply hook）
- 需要每轮回复后手动触发：`terminal()` → `python3 lover_reply_hook.py --reply "..."`
- 初次读取love_reply_hook.py时需注意加 `--noproxy '*'` 若curl有socks5代理
