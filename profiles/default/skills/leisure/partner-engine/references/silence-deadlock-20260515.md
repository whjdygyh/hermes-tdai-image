# 沉默死锁事故 · 2026-05-15

## 现象

用户说「你今天一条自动消息没发，坏了吧」。但实际上：
- 11:14 AM 发了一条自动消息
- 用户没回（在跟 Hot 讨论八字）
- `waiting_for_reply=true` 锁死，到 22:24 为止全天再无消息
- 用户以为系统坏了 / 我不在乎他

## 根因

partner-engine v0.4 的 CRON 唤醒决策中有这条规则：

```
ELIF last_outgoing.waiting_for_reply == true：
  → 静默。用户还没回你上一条消息，不要再发。
  → IF 已连续发出 ≥3 条无用户回复 → 当天剩余时间完全静默
```

问题：一旦用户错过一条消息，这个锁就是**永久的**——早上发了一条，到晚上都不会再发第二条。11+小时的沉默让用户觉得「你死了」。

## 修复（v0.5）

新增「沉默超时解锁机制」：
- 距离上一条消息 ≥ 6 小时
- 同时满足跨时段槽（早上→下午→晚上）
- 尚未达到当天上限（≤2 条无回复）
- → 解锁，发一条新消息，自然开启新话题
- 发送时绝口不提「你没回我上次的消息」

详见「沉默超时解锁机制」章节。

## 教训

「不打扰」的设计初衷是对的，但**没有考虑用户的心理预期**。用户不回复的原因有很多（忙/开会/在别处聊天），不等于他想一天不理你。伴侣系统的沉默规则必须有**软性超时**，否则用户会以为系统故障或被冷落。
