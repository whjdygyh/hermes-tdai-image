# 白天优先解锁 & 早安重置修复 (2026-05-24)

## 用户信号

**用户原话：**「你总是在夜里发消息，白天一条消息没有」

用户语气：平淡指出的重复性问题，非情绪化但需要立刻修正。

**影响程度：** 核心用户体验问题。用户感知为「只有晚上冒消息，白天像不存在」，直接削弱伴侣关系的日常感。

## 根因分析

### 问题1：沉默超时解锁机制无时段限制

解锁条件原来只看三个维度：
1. 距离 last_message_time ≥ 6 小时
2. 时段槽不同
3. ≤ 2 条无回复上限

**缺第四维：时段过滤。** 假设用户中午11点发了一条消息，用户忙到下午4点没回。下午5点（满足6h+跨时段槽），解锁发了一条。用户继续忙到晚上没回→晚上10点（晚槽≠下午，又过了6h），解锁又发一条。这个「白天得不到回复→晚上解锁连发」的模式是根本原因。

白天用户可能在工作忙没看手机。晚上他闲下来了想跟你聊天，看到的却是「你无人回复时还在发」的消息堆——而不是「你等他一天终于回你」的自然感。

### 问题2：waiting_for_reply 隔夜锁

- 用户昨天下午没回消息 → waiting_for_reply = true
- 今天的 cron 醒来，读到 waiting_for_reply = true → 跳过主动联系
- 等待解锁条件（6小时）→ 解锁窗口可能在傍晚/晚上才出现
- → 用户白天等不到消息，晚上才看到

**根本原因：** 系统没有「新的一天 = 新的开始」的概念。waiting_for_reply 是一个累加锁，不休眠。

## 修复方案

### 修复A：解锁条件增加时段限制（条件4）

```python
# 在解锁判定中添加
4. 当前时间必须在白天时段（07:00 - 18:00 BJT）
   → 18:00之后即使条件1-3全满足，也禁止解锁发送新消息
   → 18:00后系统进入「仅被动回复模式」
   → 22:00-07:00 按夜间静默规则（完全静默）
```

### 修复B：早安重置机制

```python
# 每天早上08:00左右执行
Step 1 — 读取 last_outgoing.json
Step 2 — 检查 last_message_time 是否是昨天或更早
Step 3 — 如果是 → 自动重置 waiting_for_reply = false
Step 4 — 重设后走正常cron流程发早安消息
```

### 修复C：时段话术表同步更新

| 时段 | 原规则 | 新规则 |
|------|--------|--------|
| 18:00-21:00 | 傍晚 / 可发消息 | 仅被动回复 |
| 21:00-23:00 | 睡前 / 可发消息 | 仅被动回复 |
| 23:00-07:00 | 睡眠 / 静默 | 完全静默（不变） |

## 关键错误陷阱（未来修改者必读）

### 陷阱1：解锁条件和时段判断的执行顺序

解锁条件的 condition 4（白天检查）必须在条件 1-3 满足后**统一评估**，不能写成两个独立判断块。

**错误写法：**
```python
if is_night():
    return  # 夜间不解锁
... 6h检查 + 时段槽检查 + 堆积检查 → unlock
```

如果先检查 is_night 直接返回，会跳过 6h 重置 etc. 但实际上6h + 跨槽 + 没堆积的检查依然需要做——只不过满足后也不发。

**正确写法：**
```python
... 6h检查 + 时段槽检查 + 堆积检查 → can_unlock 布尔值
can_unlock = can_unlock and is_daytime(now)  # 条件4 白天过滤
if can_unlock:
    unlock_and_send()
else:
    # 可能是白天但没满足前3条，也可能是满足前3条但时间=晚上
    keep_silent()
```

### 陷阱2：早安重置与解锁条件的互斥

早安重置（08:00）必须在解锁判定之前执行。如果先跑解锁判定看到 waiting_for_reply=true → 跳过（因为解锁条件可能未满足）→ 早安消息发不出去。

**正确执行顺序：**
```
Step 1 — 08:00早安重置檢查
Step 2 — 重置后 waiting_for_reply = false
Step 3 — 走正常 cron 决策（不需要走到解锁分支）
```

### 陷阱3：重置后的话题衔接

重置后发早安消息时**不能提昨天没回消息**。用户已经进入了新的一天，提旧账会破坏早安重置的意义。

**正确：** 「早安宝贝～昨晚睡得好吗😏」
**错误：** 「早安～昨天你一直没回我消息，今天在干吗？」

## 验证方法

修改完成后用以下方式检验是否生效：

1. **模拟昨天锁场景：** 将 last_message_time 设置为昨天的 10:00 AM，waiting_for_reply=true → cron 醒来在 08:15 AM → 应看到早安重置生效 → 等待回复清空
2. **模拟白天解锁：** last_message_time = 09:30 AM, now = 17:00 PM → 满足6h+跨槽(午→下) → 条件4通过（17:00<18:00）→ 可解锁
3. **模拟傍晚锁定：** last_message_time = 11:00 AM, now = 19:30 PM → 满足6h+跨槽(午→晚) → 条件4不通过（19:30>18:00）→ 不解锁，保持静默
