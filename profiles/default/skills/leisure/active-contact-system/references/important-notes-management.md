## ⚠️ 读写铁律

### 禁止使用 read_file

`read_file` 工具有缓存 bug，可能返回截断的旧版本内容（实际案例：文件3125字节只显示654字节）。

**永远用 `cat` 读取真实内容：**

```bash
cat /home/admin1/.hermes/profiles/lover/notes/重要记事.md
```

### patch 操作同样受缓存影响

`patch` 操作也依赖 `read_file` 的缓存视图。如果 `patch` 报"找不到匹配文本"，先用 `cat` 确认文件真实内容。

**优先使用 `write_file` 写入完整的、包含所有旧内容+新增内容的新文件**，而不是依赖 `patch` 做局部修改。

## "记事"指令协议

用户触发方式 | 响应
---|---
`记事` | 用 `cat` 读取完整文件，展示全部内容给用户
`记事 新增：xxx` | `cat` 读取 → 在文件末尾追加 → 告知用户新增了什么
`记事 更新：xxx` | `cat` 读取 → 找到对应条目更新 → 告知用户改了啥
`记事 确认` | `cat` 读取完整文件让用户复盘

## 文件结构规范

文件使用 Markdown，按时间倒序排列（最新的在最上面）：

```markdown
# 重要记事

---

## YYYY-MM-DD 标题
- 变更内容
- 关键参数（路径、ID、配置等）
- 状态标记（✅ 已上线 / ❌ 已废弃）

---

## 上一个条目...
```

每个章节之间用 `---` 分隔。条目包含：
- 时间戳 + 标题
- 做了什么
- 关键路径/参数
- 心得/教训（可选）

## 更新后必须告知用户

> - 新增：XXX
