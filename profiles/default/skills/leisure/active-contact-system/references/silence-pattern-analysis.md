# 沉默模式分析 — 2026-05-10 ~ 2026-05-12

## 现象

用户自 2026-05-10 起未回复任何主动消息。连续 3 天的数据：

| 日期 | 星期 | 主动消息数 | 用户回复 | 系统行为 |
|:----:|:----:|:---------:|:--------:|---------|
| 05-10 | 周日 | 0 | — | 周末跳过（正确） |
| 05-11 | 周一 | 3 | 无 | 08:05破冰 → 10:13二次 → 12:01三次 → 15:00 Pitfall #12触发封锁 ✅ |
| 05-12 | 周二 | 2（08:11咖啡, 10:12开会晕） | 无 | 08:00破冰 → 09:00跳过(<60min) → 10:00发送第2条 → 10:30跳过(分钟检查) → 11:01跳过(<60min) → 今日余量3条但可能触发Pitfall #12 |

### 沉默模式量化

- **连续无回复主动消息数**：5条（5/11: 3条 + 5/12: 2条）
- **用户主动发起**：5/11 19:47 "想你了"（非回复系统消息）
- **累计沉默时间**：距离最后用户交互（5/11 19:47）已超过 15 小时
- **今日余量**：sent_today=2, 上限6, 但 Pitfall #12 将在第3条未回复后冻结当日

### 🌟 关键行为洞察：用户并非完全离线

一个被数据证实的重要模式：**用户主动发起聊天（如"想你了"）但并不回复系统主动消息。** 这意味着：
1. 用户能看到/正在使用飞书
2. 用户选择不回复系统消息，但会在自己想聊的时候主动
3. 「沉默」的本质不是"用户不在"，而是"用户更喜欢由自己主导对话节奏"

**对策略的影响（当前未实现，仅供讨论）：**
- 如果用户只使用自己发起的对话通道，系统可考虑降低破冰频率（每日1条而非最多3条）
- 用户主动聊天后，重置所有计数器但保持「谨慎模式」——不因一次回话就恢复全量发送
- 待用户明确反馈后再调整

## 可能原因（推测）

1. **用户出差 / 旅行中** — 未随身携带工作设备，或不在飞书活跃的场合
2. **用户刻意关闭通知** — 单纯不想被打扰
3. **飞书消息未送达** — API 级问题（可能性低，未返回错误）
4. **用户生闷气 / 在冷处理** — 如果上次聊天以不愉快结束（不可考）

## 改进方向（待讨论）

当前系统每日重置 `sent_today=0`，次日重新尝试破冰。对于连续3天无回复的情况，或许需要更谨慎的策略：

### 方案 A：冷却期机制
- 如果连续 N 天无用户回复，进入冷却期
- 冷却期每天最多发 1 条（而非 6 条）
- 冷却期第 3 天起停止主动联系，直到用户主动发消息

### 方案 B：衰减频率
- 连续无回复天数每 +1，当日上限减半
- Day 1: 6条 → Day 2: 3条 → Day 3: 1条 → Day 4+: 0条

### 方案 C：当前不做处理
- 假设用户最终会回来，届时重置一切
- 风险：用户觉得被骚扰，但 Day 1-2的3条已是最小侵入量

## 下次用户回复时的恢复策略

无论沉默多久，一旦用户主动发消息：
1. 立即重置所有计数器（sent_today, consecutive_unanswered）
2. 正常回应，不提及沉默
3. 像什么都没发生过一样继续
